# jpFun 语法规范

## 1. 核心思想
完全函数式，但提供语法糖供快捷调用。

## 2. 基本类型
- 字符串，用引号包裹。如果不会引起歧义，可以不加引号
- bool值，只能为 `true` / `false`
- 数字
- 内容，用 `{}` 包裹。如果不会引起歧义，可以不加括号
- 标签


“内容”的基本元素：
- 函数调用：`@func(...)`，函数名后必须立即接`(`，参数使用逗号分隔，参数可以是(2)中提到的四个类型
- 标签：`XXX@label`，不按要求的函数调用都会被理解为label，用于关系型函数的参数

> 歧义：比如字符串为 `"123,456"`，在函数调用时，逗号是参数的分隔符，如果没有引号，则会认为传递了两个数字，引发歧义。因此必须用引号分割

注释：`%` 直到行末

## 3. 主要函数
- `@1(tonality)` 修改调性 如 `1=C` 作用域为当前时间之后
- `@bpm(number)` 修改bpm 作用域为当前时间之后

- `@set(key=value, ...)` 修改函数的缺省值，作用域为当前 `{}`


- `@line(...)`


- `@voice({}, 音轨名, 名称1={}, 名称2={}, ...)`
    语法糖:
    ```
    N(音轨名): ... (note等元素)
    L(名称1): ... 歌词1
    L(名称2): ... 歌词2
    ```
    起始条件1: `N(xxx):`。终止条件：该层级的换行符或结束。可以有大括号，大括号内随意换行
    起始条件2: `L(xxx):`。终止条件：该层级的换行符或结束。如果上一个不是voice就报错

- `@over(...)`

- `@note(...)` / `@n(...)`
    - 音名：`A B C D E F G`
    - 可见休止：`0`（`Z` 等价 `0`）
    - 隐形占位：`8`
    - 升降符：`#` `b` `n`（可重复，如 `bb`、`###`）
- `@div(...)` / `@/(...)`
    - 语法糖：直接在后面加`/`，可多个
- `@dot(...)` / `@.(...)`
    - 语法糖：直接在后面加`.`，可多个
- `@dash()` / `@-()`
    - 语法糖：直接在后面加`-`，可多个
- `@bar(...)` 创建一个小节线 时间线上强制对齐

### 关系型
参数类型为标签，如果不传，则默认取最后两个注册的节点(比如note)
- `@beam(...)`：将传参的减时线相连
- `@tie(...)`：给传参加上连音线使之相连

语法糖实例：
- `@bar(C1 C3)` -> `@bar({ C1 C3 })`
- `{...}/` -> `@div({ ... }, 1)`
- `{...}.` -> `@dot({ ... }, 1)`
- `-` -> `@dash()`
- `--` -> `@dash() @dash()`
- `C1 C3# |` -> `@bar({@note(C,1), @note(C,3,#)})`


## 4. 参数规则

命名参数取值顺序固定：

1. 显式传参
2. 上下文变量（`@set`）
3. 默认值

## 5. 错误策略

- 未注册函数：`E_UNKNOWN_FUNCTION`，忽略该调用并继续
- 结构破坏（未闭合等）：`fatal`
